# Games Shakedown — 2026-06-05

Read-only friction-find pass across the eight games in the Lionade suite (Roardle, Timeline, Flashcards, Sketchy Subjects, Bluff Trivia, Pardy, Resume Coach, Mastery Mode). Motivated by today's WhatsApp tester report from SantiSanty (Roardle input-stuck after invalid word) — goal is to catch the next batch before any reach a user. Found **48 findings**: 4 P0 tester-blockers (one of which is iOS-fatal for Timeline), 28 P1 friction items spread across the eight games, and 16 P2 polish. A handful of cross-game patterns (no `autoCorrect` on any text input, daily limit is localStorage-only) are worth a refactor pass and called out at the bottom.

## 🚨 Cross-game patterns (worth a refactor pass)

1. **No iOS keyboard hints on any text input.** Sketchy guess input, Bluff fake input, Pardy answer textarea, Resume Coach SocraticBubble textarea, Mastery socratic textarea, Mastery landing parse textarea — none have `autoCorrect="off"` / `autoCapitalize="off"` / `spellCheck={false}`. iPhone users will type "Photosynthesis" capitalized when the answer is lowercase; iPad autocorrect can replace words silently. **One-line fix per file** but worth doing systematically. (6 files.)

2. **Daily-limit cap is client-only.** `app/games/page.tsx:87-100` (`getDailyPlays` / `incrementDailyPlays` on localStorage) is the ONLY guard against the daily cap. Server `/api/games/reward/route.ts` caps the per-call magnitude but does not track per-day count. A bad-faith user can grind 50 Fangs × unlimited calls per day. **Server-side rate-limit via `daily_game_plays` table** is the only honest fix and matches today's audit P0 trust-issue theme.

3. **HTML5 `draggable` API is touch-broken on iPhone.** Timeline is the only game that uses it, but if any future game (e.g. a card-sorting Pardy V2) reaches for it, the same iPhone-blocked outcome applies. Establish "no native drag-and-drop on touch devices" as a standing rule, and ship a `<DraggableRow>` component that abstracts pointer events.

4. **Refresh-mid-round survives Roardle + Timeline + Mastery, but NOT Sketchy / Bluff / Pardy / Flashcards / Resume Coach.** Of those, Pardy is the most painful (long sessions, lots of Fangs earned). Pattern: a `game_session_state` table keyed `(user_id, game_type)` with a generic `state JSONB` column would cover all of them. Roardle + Timeline already use exactly this (`quiz_session_state`); generalize and adopt.

5. **Modal dismiss inconsistency.** Pardy modal: ESC ✓, backdrop ✗. Mastery LimitPaywall: ESC ✓, backdrop ✓. Mid-game invite modal in Sketchy/Bluff: not verified but worth confirming. Adopt a standard `<Modal>` component with ESC + backdrop + focus-trap baked in (today's audit also called this out via `window.confirm()` findings).

6. **Dead buttons are the highest-trust issue and appear in TWO places in this audit.** Pardy's Skip (P0 #3) and Mastery LimitPaywall's "Archive" / "Upgrade" (P1) both look real but no-op. Today's audit P0 #4 was the Profile Delete Account button. **A scan for `onClick={() => {}}` and `onClick={onClose}` on visible CTAs across the app would catch the rest** in one pass.

## Summary for triage

Audited eight games end-to-end: Roardle, Timeline, Flashcards, Sketchy Subjects, Bluff Trivia, Pardy, Resume Coach, Mastery Mode. Found 48 findings: 4 P0 (the highest-impact is Timeline being literally unplayable on iPhone because it uses HTML5 native `draggable`, which Safari does not fire from touch — same tester profile as today's Roardle bug), 28 P1, 16 P2. The most consequential P0 after Timeline is Pardy's Skip button, which closes the modal without marking the tile attempted — so the FINAL TALLY screen is unreachable unless the user answers every single tile, blocking game-over for any player who wants to skip a clue. The other two P0s are Library-mode Roardle (target word may not be in WORD_BANK so submitting the correct answer fires "Not in word list," exact same input-stuck class as today's bug) and Library-mode Flashcards (NaN divide-by-zero when the PDF has no vocab).

Six cross-game patterns are worth a refactor pass. The biggest is that no text input in any game has `autoCorrect="off"` / `autoCapitalize="off"` / `spellCheck={false}` — iPhone auto-caps "fermentation" and can silently substitute words mid-bluff.
