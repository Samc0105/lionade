## Deliberate No-Row Decisions (perf/architecture refactors)

These web changes intentionally have **no parity row** — recorded here so a future audit doesn't misread the absence as missed tracking:

- **2026-05-25 — Cache + SWR Phase A — tab-switch refetch fix (both platforms staged, awaiting user greenlight to push/build).** Web ran a surgical 5-change pass on top of the 2026-05-17 SWR navigation refactor: (1) global SWR defaults `revalidateOnFocus: true → false` + `dedupingInterval: 5s → 60s` (`lib/swr-config.ts`); ~24 explicit per-hook `revalidateOnFocus: true` overrides preserved on the freshness-sensitive surfaces (Navbar notifications, `useUserStats` Fangs balance, Social unread badges, ClockIn, StreakRevive, DailySpin, etc.). (2) Shop inventory raw `useEffect → useSWR("shop-inventory/" + uid, ...)` + `mutateInventory()` from purchase/equip (`app/shop/page.tsx`). (3) Compete leaderboard raw `useEffect → useEloLeaderboard(5)` (`app/compete/page.tsx`, uses existing `lib/hooks.ts` hook). (4) `/api/notifications` poll deduped — Navbar + Social both now keyed on `notifications/${user.id}`, single 15s poll + single realtime channel invalidates the shared key (`components/Navbar.tsx`, `app/social/page.tsx`). (5) Dashboard `POST /api/bounties/rotate` gated by `localStorage["bounties-last-rotation"]` 1h timestamp instead of firing per-mount (`app/dashboard/page.tsx`). This is a **perf / data-fetch-cadence** change with **zero user-facing feature surface** — same precedent as the 2026-05-17 entry below. **iOS team is running an equivalent Phase A in parallel** (different libraries, same goal: kill tab-switch revalidation storms); the 2026-05-13 audit already assessed the iOS data layer as exemplary on the raw-fetch dimension. Both platforms staged — web has a local commit (NOT pushed); iOS code ready (NOT yet built into a TestFlight). Awaiting Sam's review + greenlight before push/build. Chain: `admin` (web orchestrator) → in-place implementation (5 files) → type-check pass → `quality-qa-tester` (5 happy-path + edge cases) → `quality-code-reviewer` clean → `quality-docs-writer` (CHANGELOG + vault Daily/2026-05-25.md + this row). **Update (same day, separate commit): Phase A retrofit audited the ~24 explicit `revalidateOnFocus: true` per-hook overrides that the initial Phase A had left in place. Per Sam's confirmed keep-list (`useUserStats`, `useStreakInfo`, `usePlan`, Navbar+Social notifications, live mastery session), 17 overrides stripped + 7 kept across 16 files. 3 borderline strips flagged for human review (ClockIn, StreakRevive, DailySpin — all retain `refreshInterval: 60_000` so freshness preserved). Type-check clean, QA + code review clean. Local commit, NOT pushed.** **Phase B (same day, separate commit) — shared persistence layer + iOS hydration gate.** vp-ios shipped the canonical `StorageAdapter` interface + `createPersistedSwrProvider` factory in `@lionade/core/cache/storage.ts` (consumed by both platforms; admin's web team wires the localStorage side, vp-ios wires the iOS AsyncStorage side). On iOS, `lib/swr-config.ts` was refactored to consume the factory and now exports `swrCacheReady: Promise<void>` — `app/_layout.tsx` gates `SplashScreen.hideAsync()` on BOTH fonts AND that promise, so the first paint of any signed-in user already has hydrated cache (kills the cold-start "pop, pop, pop" of skeleton flash).
